## 1. The One-Sentence Story

> Benchmarked deterministic, ensemble, and diffusion-based neural surrogates for 2D heat-equation fields under exact and noisy boundary observations, testing when generative models provide better-calibrated posterior uncertainty than simpler baselines.

This describes what the project *does*, not what it *finds*. The results may show DDPM outperforms the ensemble, or they may not. Either finding is valid. The project is a benchmark, not an advocacy piece.

## 2. What This Project Is (and Is Not)

This project **compares** four classes of neural surrogate for PDE solution operators:

1. A deterministic U-Net regressor (point estimate)
2. A Fourier Neural Operator (operator-learning baseline)
3. A deep ensemble of U-Net regressors (probabilistic, non-generative)
4. A conditional DDPM (probabilistic, generative)

It evaluates them with **physics-aware metrics** that go beyond pixel-level accuracy, and tests **when** a generative model adds value over simpler probabilistic alternatives.

**What the project does not presume:**
- It does not assume DDPM will outperform the ensemble. That is an empirical question.
- It does not claim diffusion models are better PDE solvers. The FD solver is exact.
- It does not claim the uncertainty estimates are general-purpose physics uncertainty.

## 3. The PDE Problem

### Steady-state heat equation (Laplace)

```
nabla^2 T(x,y) = 0    on Omega = [0,1]^2
T = g(x,y)             on dOmega (Dirichlet BCs)
```

### The solver

5-point finite difference Laplacian. Build the sparse matrix once, LU-factorize once, then back-substitutions per sample (~0.5ms each).

### Boundary condition generation with corner consistency

**Step 1:** Sample 4 corner values from Uniform(-1, 1).

**Step 2:** Generate each edge profile constrained to match its endpoint corners via x(1-x) envelope.

**BC perturbation types (5 families):**

| Type | Formula | Parameters |
|------|---------|------------|
| Sinusoidal | A*sin(n*pi*x) | A in [0.5, 2.0], n in {1,2,3,4} |
| Random Fourier | sum_k a_k*sin(k*pi*x), K=5 | a_k ~ N(0, 1/k^2) |
| Gaussian bump | A*exp(-(x-mu)^2/2*sigma^2) | A in [0.5, 3.0], mu in [0.3, 0.7] |
| Piecewise constant | Smooth step, 2-3 levels | Levels in [-2, 2] |
| Linear (no perturbation) | 0 | baseline only |

**Piecewise clarification:** The "piecewise constant" OOD family uses smooth step transitions (via tanh with width=0.05, attenuated by the x(1-x) endpoint envelope), not true discontinuities. The OOD signal is the step-like interior structure, which is absent from all training BC families.

### Dataset sizes and splits

| Split | Size | BC families | Purpose |
|-------|------|-------------|---------|
| train | 40,000 | sinusoidal, Fourier, bump, linear | Main training |
| val | 5,000 | sinusoidal, Fourier, bump, linear | Hyperparameter tuning |
| test_in | 5,000 | sinusoidal, Fourier, bump, linear | In-distribution evaluation |
| test_ood | 1,000 | **piecewise only** | Out-of-distribution generalization test |

## 4. The Three Phases

### Phase 1: Deterministic Baselines (Days 1-5)
- FD solver + dataset generation (including held-out piecewise split)
- BC conditioning with 8-channel encoding
- U-Net regressor (deterministic baseline)
- FNO (operator-learning baseline)
- Physics-aware evaluation metrics
- Shippable as standalone neural operator benchmark after Day 5

### Phase 2: Probabilistic Models + Noisy BCs (Days 6-10)
- Deep ensemble of 5 U-Net regressors (probabilistic non-generative baseline)
- Conditional DDPM (probabilistic generative model)
- Noisy/sparse BC observation regime
- Posterior evaluation: coverage, CRPS, functional-level UQ
- Matched-sample comparisons (5v5) as primary, full-sample (5v10) as secondary
- Main finding: does DDPM earn its computational cost over ensembles?

### Phase 3: Physics-Informed Regularization (Day 11, stretch)
- PDE-residual loss for DDPM with per-sample timestep-dependent weighting

## 5. Conditioning Design (8-Channel Standard)

All models use the same 8-channel conditioning tensor: 4 value channels + 4 mask channels = (8, 64, 64).

- Channels 0-3: BC values (top, bottom, left, right), broadcast along perpendicular axis
- Channels 4-7: observation masks, broadcast along perpendicular axis
- Phase 1: all masks = 1.0
- Phase 2: masks have 1.0 at observed positions, 0.0 at interpolated

## 6. Observation Regimes (Phase 2)

| Regime | Points/edge | Noise sigma | Description |
|--------|------------|-------------|-------------|
| exact | 64 | 0 | Phase 1 baseline |
| dense-noisy | 64 | 0.1 | All points, moderate noise |
| sparse-clean | 16 | 0 | Few exact observations |
| sparse-noisy | 16 | 0.1 | Most realistic scenario |
| very-sparse | 8 | 0.2 | Extreme uncertainty |

All Phase 2 models trained from scratch (no Phase 1 warm-starting).

## 7. Models

- Model 0: FD solver (oracle reference)
- Model 1: Conditional U-Net regressor (~5M params)
- Model 2: FNO (~2M params)
- Model 3: Deep ensemble of 5 U-Net regressors
- Model 4: Conditional DDPM (T=200, linear beta schedule)
- Model 5: DDPM + physics regularization (stretch, per-sample timestep weighting)

## 8. Evaluation

**Sample count fairness:** 5 ensemble members vs 5 DDPM samples (primary). DDPM(10) as secondary.

**Pre-committed interpretation rule:** DDPM only wins if it improves at matched-5. Otherwise reported as higher-cost tradeoff.
